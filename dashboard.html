🔗 Arquitetura do sistema
[ESP32] ──POST /dados (a cada 5s)──► [Python Flask + SQLite] ▲ │ └──── comando (bomba, servo, etc.) ◄────────┘ │ [Dashboard HTML] ──GET /status, /historico───►┘

O ESP32 envia os dados dos sensores ao backend Python a cada 5 segundos via HTTP POST. O backend salva no banco SQLite e retorna qualquer comando pendente. O dashboard busca o status atual e o histórico independentemente, lendo direto do banco.

Arquivos do projeto: aquamaster_esp32_v3.ino → firmware do ESP32  |  aquamaster_api.py → backend Python  |  dashboard.html → este painel  |  aquamaster.db → banco SQLite (criado automaticamente)

📊 Cards de status

🌡️ Temperatura — Leitura do sensor DS18B20 (GPIO 4). Mostra a temperatura atual em °C e indica se está dentro da faixa configurada nas Configurações. Valor -99 indica sensor desconectado.

💧 pH da Água — Leitura do módulo PH-4502C (GPIO 32). A faixa ideal para Beta é 6.5–7.5. Usa calibração dual-slope com 3 pontos (pH 4, 7 e 10) para maior precisão em toda a faixa. Consulte o card de Calibração para o processo completo.

⚡ Bomba d'Água — Estado atual do relé (GPIO 18). Liga/desliga no ciclo automático (15 min ligada / 5 min pausa por padrão no firmware).

🍽️ Alimentador — Estado do ciclo de 7 dias. Ciclo ativo: aguardando próximo dia. Aguardando recarga: ciclo completo, coloque ração e clique "Iniciar ciclo".

📈 Gráficos e filtros de tempo

Os gráficos mostram o histórico de temperatura e pH. Use os botões 1h / 6h / 24h para escolher a janela de tempo. Clicar num botão busca os dados imediatamente, mesmo com a atualização pausada.

Como funciona o downsampling: com 5s de polling, 24h geram ~17.000 pontos. O sistema reduz automaticamente — 1h mantém todos os pontos, 6h usa 1 a cada 25s, 24h usa 1 a cada 100s — entregando sempre gráficos fluidos.

Se a janela selecionada for maior que o período de dados disponíveis, o gráfico mostrará apenas o que existe no banco.

⏸ Pause a atualização automática antes de analisar os gráficos para evitar que os dados sejam recarregados enquanto você observa.

🔄 Ciclo de alimentação — 7 dias

O disco do alimentador tem 7 compartimentos. O servo gira para a posição de cada compartimento uma vez por dia, no horário configurado em ⚙️ Configurações → Horário de alimentação diária (padrão: 08:00h, fuso UTC-3).

Sequência: Dia 1 → Dia 2 → … → Dia 7 → aguarda 10s → retorna ao Home (0°) → entra em modo "Aguardando recarga". Coloque ração nova e clique Iniciar ciclo para recomeçar.

Posições padrão: Dia 1 = 25°, Dia 2 = 50°, Dia 3 = 75°, Dia 4 = 105°, Dia 5 = 135°, Dia 6 = 160°, Dia 7 = 180°. Ajuste individualmente em ⚙️ Configurações → Posição do servo por dia. Ao salvar, as posições são sincronizadas com o ESP32 automaticamente.

Para testar: use o botão Alimentar agora no card do Alimentador — o ESP32 aciona o servo imediatamente, sem esperar o horário agendado.
📐 Servo motor — ajuste físico

O servo SG90 (GPIO 13) vai de 0° a 180°. O gauge em arco mostra a posição atual lida do ESP32. O badge indica se é Home ou qual dia corresponde ao ângulo atual.

Slider 0–180°: arraste e clique "Mover" para uma posição exata.

Botões de preset: Home (0°) e Dia 1–7 com os ângulos configurados. O botão correspondente à posição atual fica destacado.

Campo manual: digite qualquer valor entre 0 e 180° para ajuste fino. Use enquanto posiciona o disco físico do alimentador.

Movimentos via dashboard não alteram o estado do ciclo automático — servem apenas para ajuste físico.

🕐 Agenda de alimentação

Defina horários fixos (ex: 08:00, 12:00) para disparar alimentar_agora automaticamente. A agenda é verificada a cada 10 segundos pelo dashboard e dispara o comando via backend.

Limitação importante: a agenda funciona apenas enquanto a aba do dashboard estiver aberta no navegador. Para agendamento independente do browser, será necessário adicionar um scheduler ao backend Python (fase futura).

Use o toggle ativo/pausado em cada horário para habilitar ou desabilitar sem remover o horário.

🧪 Calibração do sensor de pH — Processo completo (dual-slope)

Por que dual-slope? O sistema usa 3 soluções tampão (pH 4, 7 e 10) e calcula dois slopes independentes — um para a faixa ácida (4–7) e outro para a alcalina (7–10). Isso elimina o erro acumulado de uma reta única e entrega leituras precisas em toda a faixa de interesse para aquários.

Materiais necessários: solução tampão pH 4.0, pH 7.0 e pH 10.0 (kits de calibração vendidos juntos), água destilada para enxágue entre mergulhos, e um copo pequeno para cada solução.

🔒 ANTES DE COMEÇAR — Cuidados com o eletrodo

O eletrodo deve ser sempre armazenado na solução KCl (o líquido que veio dentro do capuchon rosqueado). Nunca guarde em água destilada — resseca a membrana e inutiliza o sensor. Retire a proteção rosqueada apenas na hora de calibrar ou medir.

📋 ETAPAS DE CALIBRAÇÃO

① Abra o Arduino IDE → Monitor Serial com velocidade 115200 baud. Certifique-se de que o ESP32 está ligado e conectado ao computador. Selecione Ambos NL & CR (ou Both NL & CR) no rodapé do monitor.
② Entre no modo calibração: digite exatamente Calibrar Sensor PH e pressione Enter. O ESP32 começará a exibir a tensão no GPIO 32 a cada 1 segundo. O sistema continua controlando a bomba e o alimentador normalmente.
③ Calibrar pH 7.0 (ponto neutro): Enxágue o eletrodo com água destilada e mergulhe na solução pH 7.0. Aguarde a tensão estabilizar (20–30 segundos). Anote o valor de Tensao GPIO32 exibido. Em seguida envie: neutro 2.XXXX — substituindo pelo valor anotado.
Exemplo: neutro 2.5123
④ Calibrar pH 4.0 (tampão ácido): Enxágue o eletrodo com água destilada, mergulhe na solução pH 4.0 e aguarde estabilizar. Anote a tensão e envie: acido X.XXXX
Exemplo: acido 3.2950
O firmware recalcula o slopeÁcido automaticamente e exibe o novo valor.
⑤ Calibrar pH 10.0 (tampão alcalino): Enxágue o eletrodo com água destilada, mergulhe na solução pH 10.0 e aguarde estabilizar. Anote a tensão e envie: alcalino X.XXXX
Exemplo: alcalino 2.0070
O firmware recalcula o slopeAlcalino automaticamente.
⑥ Verificação e ajuste fino (opcional): Mergulhe o eletrodo em uma das soluções tampão novamente e veja se o pH calculado confere. Se houver uma diferença pequena constante em toda a faixa, use offset X.XX para corrigir.
Exemplo: sensor lendo 6.85 com eletrodo no pH 7.0 → offset 0.15
⑦ Finalizar e persistir os valores: Digite Concluido e pressione Enter. O ESP32 imprimirá todos os valores finais:
tensaoNeutro, tensaoTampao4, tensaoTampao10, PH_OFFSET, slopeAcido, slopeAlcalino.
Copie esses valores para as constantes correspondentes no topo do .ino e regrave o firmware — assim a calibração persiste após qualquer reinicialização.

⌨️ REFERÊNCIA DE COMANDOS SERIAL (115200 baud)

Calibrar Sensor PHEntra no modo calibração — leituras a cada 1s, bomba e alimentador continuam ativos.
neutro X.XXXXDefine a tensão medida na solução pH 7.0. Recalcula ambos os slopes automaticamente.
acido X.XXXXDefine a tensão medida na solução pH 4.0. Recalcula o slopeÁcido.
alcalino X.XXXXDefine a tensão medida na solução pH 10.0. Recalcula o slopeAlcalino.
offset X.XXAplica um ajuste fino constante ao pH calculado (positivo ou negativo). Use quando todos os pontos estão deslocados por um valor fixo.
ConcluidoSai do modo calibração, imprime todos os valores finais e retoma operação normal.
💡 Os valores não persistem no ESP32 após desligar — a memória SRAM é apagada no reset. Sempre copie os valores exibidos pelo comando Concluido para as constantes tensaoNeutro, tensaoTampao4 e tensaoTampao10 no início do .ino e regrave.
🔁 Quando recalibrar? O eletrodo de vidro envelhece — recalibre a cada 1–2 meses, sempre que trocar a solução KCl de armazenamento, ou se as leituras parecerem inconsistentes.
